title: "Hugging Face Credentials for Gated Models"
sidebarTitle: "HF Credentials"

---

Many popular models on Hugging Face — including Llama 3, Gemma, Mistral, and other instruction-tuned families — are published under a gated licence that requires you to accept terms of use on the Hub and authenticate with a personal read token before downloading. Odysseus Portable has a built-in token resolution pipeline that checks several locations automatically, so once you store your token in any one of the supported places it is picked up without further configuration. The Cookbook download UI also exposes a token field directly in the browser, meaning you never have to touch the terminal at all if you prefer a graphical workflow.

<Note>
  A Hugging Face token is only required for gated or private model repositories. All public GGUF models available without a login — including the vast majority of community quantisations hosted on Hugging Face — download without any credentials.
</Note>

---

## Setting Up Your Token

<Steps>
  <Step title="Create a Hugging Face Read token">
    Go to [https://huggingface.co/settings/tokens](https://huggingface.co/settings/tokens) and sign in. Click **New token**, choose the **Read** role, give it a name (e.g., `odysseus-portable`), and click **Generate a token**. Copy the token — it starts with `hf_`.
  </Step>
  <Step title="Create the odysseus/.env file">
    Inside the project folder, create a plain text file at the path `odysseus/.env`. The `odysseus/` subdirectory is the cloned Odysseus web application — place the file directly inside it, not in the project root.

    ```
    Odysseus-Portable/
    └── odysseus/
        └── .env    ← create this file
    ```
  </Step>
  <Step title="Add the token to the file">
    Open `odysseus/.env` in any text editor and add the following line, replacing the placeholder with your actual token:

    ```env
    # odysseus/.env
    HUGGING_FACE_HUB_TOKEN=hf_yourTokenHere
    ```
  </Step>
  <Step title="Restart Odysseus Portable">
    Stop the currently running instance with `Ctrl+C`, then run `start.bat` (Windows) or `./start.sh` (macOS/Linux) again. The token is loaded automatically at startup.
  </Step>
</Steps>

<Warning>
  Never commit the `odysseus/.env` file to a public git repository. It contains a secret credential. Add it to your `.gitignore` if you fork or version-control the project, and revoke the token on Hugging Face immediately if it is accidentally exposed.
</Warning>

## Alternative: Environment Variable

If you prefer not to create a file, you can pass the token as an environment variable directly at launch time:

<Tabs>
  <Tab title="Linux / macOS">
    ```bash
    HUGGING_FACE_HUB_TOKEN=hf_yourTokenHere ./start.sh
    ```

    Or export it for the shell session:

    ```bash
    export HUGGING_FACE_HUB_TOKEN=hf_yourTokenHere
    ./start.sh
    ```
  </Tab>
  <Tab title="Windows (Command Prompt)">
    ```bat
    set HUGGING_FACE_HUB_TOKEN=hf_yourTokenHere
    start.bat
    ```
  </Tab>
  <Tab title="Windows (PowerShell)">
    ```powershell
    $env:HUGGING_FACE_HUB_TOKEN = "hf_yourTokenHere"
    .\start.bat
    ```
  </Tab>
</Tabs>

## Token Resolution Order

When a Cookbook download requires authentication, Odysseus Portable resolves the token by checking sources in the following order, stopping as soon as a non-empty value is found:

<Steps>
  <Step title="Cookbook UI state (highest priority)">
    A token entered directly into the HF Token field inside the Odysseus Cookbook web interface. This is stored in `odysseus/data/cookbook_state.json` in an encrypted form and takes precedence over all other sources.
  </Step>
  <Step title="HF_TOKEN environment variable">
    The `HF_TOKEN` environment variable set in the shell or passed at launch.
  </Step>
  <Step title="HUGGING_FACE_HUB_TOKEN environment variable">
    The `HUGGING_FACE_HUB_TOKEN` environment variable, which is also the key used inside `odysseus/.env`.
  </Step>
  <Step title="~/.cache/huggingface/token">
    The token file written by the official `huggingface-cli login` command on the host system (`~/.cache/huggingface/token`).
  </Step>
  <Step title="~/.huggingface/token (lowest priority)">
    The legacy token path used by older versions of the `huggingface_hub` library (`~/.huggingface/token`). Checked last as a final fallback.
  </Step>
</Steps>

## Using the Cookbook UI Token Field

If you do not want to create any files or set environment variables, you can enter your token directly in the browser:

<Steps>
  <Step title="Open the Cookbook tab">
    Navigate to `http://127.0.0.1:7070` and click the **Cookbook** tab in the Odysseus interface.
  </Step>
  <Step title="Open Environment settings">
    Click the **Environment** or settings icon within the Cookbook panel to expand the environment configuration section.
  </Step>
  <Step title="Enter your token">
    Paste your `hf_...` token into the **HF Token** field. The token is encrypted and saved to `odysseus/data/cookbook_state.json` and will be used for all subsequent download requests in that session and beyond.
  </Step>
</Steps>

<Accordion title="Why does Odysseus need to patch the token loader?">
  The upstream Odysseus application only reads the HF token from its own internal state file by default. Odysseus Portable applies a self-healing patch to `cookbook_routes.py` at each startup that extends this lookup to also check the `HF_TOKEN` and `HUGGING_FACE_HUB_TOKEN` environment variables and both standard token file paths used by the `huggingface-cli` toolchain. This means credentials you have already configured system-wide or via the `.env` file are honoured without any extra steps.
</Accordion>
